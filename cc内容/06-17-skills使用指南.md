# Claude Code Skills 使用指南

本仓库 `D:/git/skills/` 目录下共收录 **24 个 Claude Code skills**，覆盖设计、工程开发、工作流规划、配置工具等多个领域。本文档按类别整理每个 skill 的用途、调用方式与典型场景。

## 调用方式总览

Skills 有两种触发方式：

| 触发方式 | 示例 | 适用场景 |
|---------|------|---------|
| **斜杠命令显式调用** | `/tdd`、`/diagnose`、`/handoff` | 精确指定要执行哪个 skill |
| **自然语言隐式触发** | "做个落地页"、"调试这个 bug" | 当你的描述命中 skill 的 trigger 关键词时自动激活 |

> 部分 skill 在 frontmatter 中标注了 `disable-model-invocation: true`（如 `teach`、`zoom-out`、`setup-matt-pocock-skills`），**必须用斜杠命令显式调用**。

## 一、设计 & 演示类（4 个）

### 1. axi-front-design — HTML 高保真设计稿

- **路径**：`skills/axi-front-design/`
- **作用**：扮演资深设计师，用 HTML 产出落地页、幻灯片、交互原型、动画、设计系统、信息图、移动端 mockup
- **触发词**：`/axi-front-design`、"做个落地页"、"设计海报/封面"、"做个交互原型"、"design a landing page"、"hi-fi mockup"
- **特点**：先问后做（至少问 4 个问题），给出多版本变体，规避 AI 风格俗套
- **不适用**：纯文字/逻辑代码、真实图像生成、原生 PPTX/Figma 文件

### 2. excalidraw-diagram — Excalidraw 图表

- **路径**：`skills/excalidraw-diagram/`
- **作用**：生成 `.excalidraw` JSON 文件，做能"可视化论证"的图表（流程图、架构图、概念图）
- **触发词**："画个流程图"、"做个架构图"、"visualize this workflow"
- **核心原则**：图表应该 **argue, not display**——去掉文字结构本身也能传达含义
- **品牌定制**：所有颜色集中在 `references/color-palette.md`

### 3. guizang-ppt-skill — 横向翻页网页 PPT

- **路径**：`skills/guizang-ppt-skill/`
- **作用**：生成单文件 HTML 横向翻页 PPT，提供两种风格
  - **风格 A · 电子杂志 × 电子墨水**：衬线字体 + WebGL 流体背景 + 暖色，适合人文分享、商业发布
  - **风格 B · 瑞士国际主义（Swiss Style）**：无衬线 + 网格点阵 + 克莱因蓝/柠檬黄/柠檬绿/安全橙高亮，适合科技产品、数据汇报
- **触发词**："杂志风 PPT"、"瑞士风 PPT"、"Swiss Style"、"horizontal swipe deck"

### 4. html-ppt — HTML PPT Studio

- **路径**：`skills/html-ppt/`
- **作用**：专业静态 HTML 演示文稿，**36 个主题 + 15 个完整模板 + 31 种布局 + 27 种 CSS 动画 + 20 种 canvas 特效**
- **触发词**：`/html-ppt`、"幻灯片"、"演讲稿"、"做一份 PPT"、"小红书图文"、"pitch deck"
- **特点**：一键 `npx skills add` 安装，键盘 ← → 翻页、S 键进入演讲者模式
- **与 guizang-ppt-skill 区别**：html-ppt 模板更丰富（36 主题 vs 2 风格），适合多场景；guizang 专注两种特定风格

## 二、工程开发类（6 个）

### 5. diagnose — 难 Bug 诊断循环

- **路径**：`skills/diagnose/`
- **作用**：用严格的诊断循环处理难 bug 和性能回归：**复现 → 最小化 → 假设 → 插桩 → 修复 → 回归测试**
- **触发词**：`/diagnose`、"调试这个"、"something is broken"、"性能回归"
- **核心**：**先建立反馈回路**（failing test / curl / CLI / headless browser / trace replay），没有反馈回路看代码没用

### 6. tdd — 测试驱动开发

- **路径**：`skills/tdd/`
- **作用**：红-绿-重构循环，强调通过公共接口验证行为，集成式好测试
- **触发词**：`/tdd`、"red-green-refactor"、"test-first"
- **反模式警告**：**不要一次写完所有测试再写实现**（横向切片），应纵向切片（一个测试 → 一个实现 → 循环）

### 7. prototype — 一次性原型

- **路径**：`skills/prototype/`
- **作用**：构建 throwaway 原型来回答设计问题，分两个分支
  - **逻辑/状态模型问题** → 构建交互式终端 app
  - **UI 应该长什么样** → 一个路由上生成多个完全不同的 UI 变体，通过 URL 参数切换
- **触发词**：`/prototype`、"let me play with it"、"try a few designs"、"prototype this"
- **原则**：一次性代码、一条命令跑起来、默认不持久化、不抛光、完成后删除或吸收

### 8. improve-codebase-architecture — 架构改进

- **路径**：`skills/improve-codebase-architecture/`
- **作用**：发现代码库中的"深化机会"（把浅模块变深模块），目标是可测试性和 AI 可导航性
- **触发词**：`/improve-codebase-architecture`、"重构机会"、"consolidate modules"
- **术语**：Module / Interface / Implementation / Depth / Seam / Adapter / Leverage / Locality
- **关键测试**：**Deletion test** — 想象删除这个模块，如果复杂度消失就是 pass-through，如果在 N 个调用方重现就是有价值的

### 9. migrate-to-shoehorn — 迁移到 Shoehorn

- **路径**：`skills/migrate-to-shoehorn/`
- **作用**：把测试文件中的 `as` 类型断言迁移到 `@total-typescript/shoehorn`
- **触发词**：`/migrate-to-shoehorn`、"replace `as` in tests"、"shoehorn"
- **仅限测试代码**，生产代码禁用

### 10. zoom-out — 拉高抽象层级

- **路径**：`skills/zoom-out/`
- **作用**：当你不熟悉某段代码时，让 agent 往上走一层，用项目的领域术语给你画一张相关模块和调用方的地图
- **触发词**：`/zoom-out`（**必须显式调用**，frontmatter 禁用了隐式触发）
- **适用场景**：刚接手陌生代码、需要理解某模块在整体中的位置

## 三、工作流 & 规划类（6 个）

### 11. grill-me — 质询你的计划

- **路径**：`skills/grill-me/`
- **作用**：就你的计划或设计一个接一个地追问，遍历决策树每个分支，直到达成共识；每个问题给出推荐答案
- **触发词**：`/grill-me`、"grill me"、"stress-test this plan"

### 12. grill-with-docs — 基于文档质询

- **路径**：`skills/grill-with-docs/`
- **作用**：在 grill-me 基础上，对照项目现有的领域模型（CONTEXT.md、ADRs）质询你的计划，同时**在决策定型时直接更新文档**
- **触发词**：`/grill-with-docs`、"stress-test against our docs"

### 13. to-issues — 拆分为 Issues

- **路径**：`skills/to-issues/`
- **作用**：把计划/PRD/spec 拆成可独立认领的 issues，用**tracer-bullet 纵向切片**（每片切穿 schema/API/UI/测试所有层）
- **触发词**：`/to-issues`、"convert this plan into issues"、"break down work"
- **前置**：先跑 `/setup-matt-pocock-skills` 配置 issue tracker
- **概念**：HITL（需人工）vs AFK（可全自动）切片，优先 AFK

### 14. to-prd — 生成 PRD

- **路径**：`skills/to-prd/`
- **作用**：把当前对话上下文和代码理解合成 PRD，发布到 issue tracker
- **触发词**：`/to-prd`、"create a PRD from this context"
- **特点**：**不采访用户**，只合成已知信息；先和用户确认测试 seam

### 15. triage — Issue 分类

- **路径**：`skills/triage/`
- **作用**：通过状态机驱动 issue 在 triage 角色间流转
- **触发词**：`/triage`、"triage issues"、"prepare issues for AFK agent"
- **角色**：bug / enhancement（类别）；needs-triage / ... （状态）
- **强制规则**：triage 期间发的每条评论必须以 `> *This was generated by AI during triage.*` 开头

### 16. handoff — 交接文档

- **路径**：`skills/handoff/`
- **作用**：把当前对话压缩成交接文档，供下一个 agent 接手；保存到 OS 临时目录（不污染 workspace）
- **触发词**：`/handoff "下一会话要做什么"`
- **特点**：包含"建议 skills"章节；不重复 PRD/ADR/commits 已有内容；脱敏 API keys 等

## 四、配置 & 工具类（6 个）

### 17. find-skills — 查找 Skills

- **路径**：`skills/find-skills/`
- **作用**：帮你从开放的 agent skills 生态发现并安装新 skill
- **触发词**：`/find-skills`、"how do I do X"、"find a skill for X"、"is there a skill that can..."
- **核心命令**：
  ```bash
  npx skills find [query]    # 搜索
  npx skills add <package>   # 安装
  npx skills check           # 检查更新
  npx skills update          # 更新所有
  ```

### 18. write-a-skill — 编写新 Skill

- **路径**：`skills/write-a-skill/`
- **作用**：创建新的 agent skill，包含正确结构、渐进披露、打包资源
- **触发词**：`/write-a-skill`、"create a new skill"、"build a skill"
- **流程**：收集需求 → 起草 SKILL.md（超 500 行拆分参考文件）→ 与用户 review

### 19. setup-matt-pocock-skills — 配置工程 Skills

- **路径**：`skills/setup-matt-pocock-skills/`
- **作用**：在 `AGENTS.md`/`CLAUDE.md` 里写 `## Agent skills` 块，在 `docs/agents/` 配置 issue tracker、triage 标签、领域文档布局
- **触发词**：`/setup-matt-pocock-skills`（**必须显式调用**）
- **何时跑**：首次使用 `to-issues`/`to-prd`/`triage`/`diagnose`/`tdd`/`improve-codebase-architecture`/`zoom-out` **之前**

### 20. setup-pre-commit — 配置 Pre-commit Hooks

- **路径**：`skills/setup-pre-commit/`
- **作用**：用 Husky + lint-staged 配置 pre-commit hooks，运行 Prettier、typecheck、tests
- **触发词**：`/setup-pre-commit`、"add pre-commit hooks"、"set up Husky"、"configure lint-staged"
- **自动检测**：npm / pnpm / yarn / bun 包管理器

### 21. git-guardrails-claude-code — Git 安全钩子

- **路径**：`skills/git-guardrails-claude-code/`
- **作用**：设置 PreToolUse hook，**在 Claude 执行前拦截危险 git 命令**
- **触发词**：`/git-guardrails-claude-code`、"block dangerous git"、"prevent destructive git"
- **拦截**：`git push`（含 `--force`）、`git reset --hard`、`git clean -f`、`git branch -D`、`git checkout .`、`git restore .`
- **可选作用域**：当前项目（`.claude/settings.json`）或全局（`~/.claude/settings.json`）

### 22. scaffold-exercises — 脚手架练习

- **路径**：`skills/scaffold-exercises/`
- **作用**：创建练习目录结构（sections / problems / solutions / explainers），通过 `pnpm ai-hero-cli internal lint` 并 commit
- **触发词**：`/scaffold-exercises`、"create exercise stubs"、"set up a new course section"
- **命名规范**：`XX-section-name/` → `XX.YY-exercise-name/`，每个子目录需 `readme.md`

## 五、其他（2 个）

### 23. caveman — 极简通信模式

- **路径**：`skills/caveman/`
- **作用**：超压缩通信模式，砍掉冠词、客套、对冲，**省约 75% token**，技术准确度保留
- **触发词**：`/caveman`、"caveman mode"、"talk like caveman"、"less tokens"、"be brief"
- **持久性**：触发后每次响应都生效，直到用户说 "stop caveman" / "normal mode"
- **示例**：
  - ❌ "Sure! I'd be happy to help. The issue is likely caused by..."
  - ✅ "Bug in auth middleware. Token expiry check use `<` not `<=`. Fix:"

### 24. teach — 教学

- **路径**：`skills/teach/`
- **作用**：把当前目录当作教学 workspace，通过多会话教你一个主题
- **触发词**：`/teach "你想学什么"`（**必须显式调用**）
- **workspace 文件**：
  - `MISSION.md` — 学习动机
  - `./reference/*.html` — 速查表、参考资料
  - `RESOURCES.md` — 高质量学习资源
  - `./learning-records/*.md` — 学习记录（类 ADR）
  - `./lessons/*.html` — 自包含的 HTML 课程
  - `NOTES.md` — 草稿
- **三要素**：Knowledge（来自高质量资源）+ Skills（互动课程）+ Wisdom（与他人交流）

## 注意事项

1. **Matt Pocock 工程类 skill 套件**（`to-issues`/`to-prd`/`triage`/`diagnose`/`tdd`/`improve-codebase-architecture`/`zoom-out`）共享一套项目配置，**首次使用前必须先跑 `/setup-matt-pocock-skills`**。
2. **`disable-model-invocation: true`** 的 skill（`teach`、`zoom-out`、`setup-matt-pocock-skills`）只能用斜杠命令调用，自然语言触发无效。
3. 部分英文 skill（如 `tdd`、`prototype`、`improve-codebase-architecture`）的内部说明偏向英文工程实践，但中文调用同样有效。
4. 安装新 skill：`npx skills add <github-url>`，安装在 `skills/` 目录下。
